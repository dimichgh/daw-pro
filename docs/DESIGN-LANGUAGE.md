# Design Language — "Glass Cockpit"

Dark, glassy, information-dense but calm. Every glow means something. Think: night-flight avionics meets high-end studio hardware, rendered with macOS-native polish.

## Foundations

- **Surfaces**: near-black layered glass. Base `#0B0D12`, raised panels `#12151D` with 1px `white @ 6%` hairline borders and subtle inner shadow. Use translucency (`.ultraThinMaterial`-style) sparingly, only for overlays.
- **Text**: SF Pro for labels; **SF Mono for every numeric readout** (time, tempo, dB, Hz). Readouts are "digital": mono font, slight letterspacing, colored glow.
- **Accent system** (semantic, not decorative):
  - Cyan `#3EE6FF` — playback/position/active state
  - Amber `#FFB84D` — record/warning/clipping-adjacent
  - Red `#FF4D5E` — clip/overload/destructive
  - Green `#5DFF9F` — healthy signal, success
  - Violet `#B48CFF` — AI/generated content (everything AI-touched carries violet)
- **Glow recipe**: color at full opacity core + same-color shadow (radius 6–10, opacity 0.5–0.7) + faint wide bloom (radius 20, opacity 0.15). Glow intensity may encode magnitude (meter brightness follows level). Never glow static chrome.

## Components

- **Meters**: segmented LED-style bars (Canvas-drawn), green→amber→red zones, peak-hold tick that decays, glow tracks level. 60fps via `TimelineView`.
- **Transport**: oversized readout cluster — bars.beats + SMPTE time in mono cyan, tempo editable inline, glowing play/record buttons with state halos.
- **Knobs/faders**: flat dark caps, thin neon value arcs/tracks; value readout appears on touch. Double-click resets. Fine-drag with ⌥.
- **Waveforms**: dual-tone (body + brighter RMS core), violet tint when clip is AI-generated.
- **Panels**: every panel has Simple and Pro density modes; Simple shows the 20% of controls used 80% of the time with plain-language labels ("Punchier" over "Ratio 4:1" — Pro mode shows both). Mode is a themed segmented chip pair (SIMPLE / PRO), the active half cyan-lit; Simple is the default.
- **Arrange timeline**: fixed pixels-per-beat (16 pt/beat, no zoom yet); one lane per track in sidebar order. Clips are rounded tinted blocks — signal-green for audio, playback-cyan for MIDI, **violet whenever AI-touched** — with a name label when width allows, selected clip brighter + glowing. Beat grid is hairlines; bar lines brighter; bar numbers in SF Mono. Playhead is a glowing cyan hairline driven by transport position (offset a view, never a per-frame Canvas redraw). A track's header carries an automation disclosure (axis-chart glyph, cyan-lit + glowing when the track has an enabled non-empty lane); opening it grows a beat-aligned breakpoint-editor row under the clip lane and a matching control panel in the sidebar (so both columns stay aligned). The editor is a Canvas neon polyline (glow recipe: wide faint bloom under a bright core) with glowing breakpoint dots and a faint dashed neutral guide (unity gain / center pan); **volume lanes track the fader's cyan, pan lanes stay neutral white** (mixer semantics — pan claims no accent), a **disabled lane renders dimmed** (its curve is inert; the fader/knob is manual). Click empty space adds a point, drag moves it with a live SF Mono value/beat readout by the cursor (dB for volume, L/C/R for pan), double-click or select+⌫ deletes; edits redraw only on interaction (no TimelineView). Target picker is volume/pan in v0 (a themed VOL/PAN chip pair, the active half cyan-lit, a dot marking which already has a lane); an enable chip (green ON / dim OFF) flips read vs. manual.
- **Clip editing** (arrange interaction layer): the arrange header carries a **grid-snap picker** (Off / Bar / Beat / 1/2 / 1/4, default Bar; **Bar follows the meter** — 3 beats in 3/4), a themed chip menu styled like the input-device chip, right-aligned under an "ARRANGE" label. Snap governs move, trim, and split (never fades — a fade is a continuous shape). A clip is **direct-manipulation**: drag the body to move, the ~6 pt left/right edge strips to trim, the top-corner grips to fade, double-click at a point to split (a context menu also offers Split at Playhead + curve toggles + Reset Gain). Fades render as **translucent** shading (never opaque — the waveform reads through) dimming the faded corner, its inner edge the fade curve (**linear = straight, equal-power = bowed**); a small triangular grip sits at each fade's inner end, faint at rest and bright on hover/selection; alt/⌥-click a grip flips its curve. A live drag shows an SF Mono beat/length/fade readout by the cursor. Per-clip **gain** is a subtle SF Mono cyan dB chip (dB readouts are cyan) shown only when non-zero or on hover/selection; drag it up/down to adjust. **Time-stretch** is the same right edge under ⌥ (option): a faint "≈" grip cue on the right edge of an audio clip brightens cyan while ⌥ is held, and ⌥-dragging it retargets the clip's timeline length (window-invariant, wired to `stretchClip`) instead of trimming — with an SF Mono "length · ratio" readout (e.g. "6.0 beats · 1.50×"); MIDI right-edge ⌥-drag stays a plain trim (stretch is audio-only). Any non-identity clip carries a persistent SF Mono ratio/pitch **badge** ("1.50×", "+3st") so stretched clips read distinct; a clip stretched **outside the 0.75–1.5× transparent band tints amber** (border + badge glow, a soft warning — never a hard block, the ratio range still spans 0.25–4×) with a tooltip naming the ratio. While the engine's offline stretch render is pending the clip body carries an **animated shimmer** (a slow tint-colored diagonal highlight sweep — the "working" cue, never a spinner); a failed render shows a **red border + glowing red dot**. All stretch geometry/threshold/formatting is headless in `DAWAppKit.ClipStretch` (the `ClipEdit` precedent). Audio clips draw a dual-tone peak **waveform** (body + brighter core, violet when AI) computed off-main and cached by file URL, windowed by `startOffsetSeconds` so a split or leading-trim shows the right slice of the source; the outline redraws only on data change (no TimelineView, no per-frame allocation). Every edit routes through one of the five store clip methods, so undo/coalescing come free and mid-play gain/fade changes stay seamless.
- **Take lanes** (arrange comping layer, M5 iii-c): a track that has take groups grows a **takes disclosure** on its header — a stacked-layers glyph, **signal-green glowing because a group exists** (outlined while open), the take-comping sibling of the automation disclosure. Opening it grows a beat-aligned takes section directly under the clip lane (above the automation row) and a matching control panel in the sidebar (both columns share `TimelineLanesView.takeLaneRowHeight`/`takeGroupHeaderHeight`, so they stay aligned — the automation-row precedent). Per group: a small header (stacked glyph + name + "N takes" + a **FLATTEN** chip, the escape hatch to ordinary clips) over **one sub-row per lane**. A lane row draws its take material **dim** (a faded peak waveform for audio, faded note pills for MIDI, windowed/positioned by the lane clip's absolute beats via the shared `ClipWaveform` idiom) with the comp's selected regions **glowing** over it (tinted rounded rects, the glow recipe — bloom under a bright core-stroke): **selected lane-range regions glow, unselected lane areas stay dim**. The **newest lane reads distinct** — a thin tint accent bar at its left edge and a bolder SF Mono name tag. **Comp painting**: drag horizontally on a lane row to paint that lane into the comp over the snapped beat range (the active `ClipSnap` grid), committing per-tick through `setCompSegments` (coalesced under `take.comp:<id>`, per-tick-safe like an automation drag); a **click without a drag selects the whole lane** (`selectTake`). On the main clip lane, each comp member carries a small SF Mono **"group · N" badge** (group name + lane count) and a thin **glowing splice line** at every internal comp boundary (a neon seam where the comp cuts from one take to the next). A member's context menu offers **Select Take N** per lane + **Flatten Group** (comp is edited via the lanes, so member clips reject the ordinary clip edits at the store); the sidebar lane rows add a per-lane select dot + delete, and a group Flatten. All geometry, paint-drag classification/snapping, the paint-override + normalize (kept inside what `setCompSegments` accepts), and the badge/splice derivations are headless in `DAWAppKit.TakeComp`/`TakeLaneGeometry`/`TakeLaneSelection` (the `ClipEdit`/`AutomationLane` precedent); the view is thin over them and every mutation routes through a store take method (undo/coalescing free). Violet is untouched here — takes are recorded/human unless a lane clip is itself AI-generated, in which case its material tints violet as always.
- **Piano roll**: Canvas grid + notes at 32 pt/beat; pitch rows high-at-top with a shaded black-key row under a white/black keyboard gutter (octave C's labeled C-1..C9 in SF Mono, middle C cyan). Notes are rounded cyan bars (violet when the clip is AI), velocity → fill opacity, selected = brighter + hairline + subtle bloom. Snap resolutions read musically: Off / Bar / Beat / 1/8 / 1/16 (default Beat). Simple mode locks to Beat with add/move/delete only; Pro adds a velocity lane (SF Mono readouts), snap picker (themed chip menu), and resize handles.
- **Mixer console**: a horizontal rack of fixed-width (132 pt) glass strips — audio/instrument channels in project order, then the visually-grouped buses (dimmer panel + a "BUSES" separator), with a wider (156 pt) cyan-accent-bordered master pinned at the right (outside the scroller). Strip anatomy top→bottom: name + color-coded **kind badge** (audio = signal-green, instrument = playback-cyan — mirroring the timeline clip colors; bus = neutral slate; **violet never codes a kind**, it flags AI-generated tracks via an AI-border + dot), Inserts (each: bypass dot glowing signal-green when passing / dim when bypassed + beginner-readable name; hosted AU shows its AU name; empty = "No inserts"; "+" adds built-ins), Sends (bus name + mono mini-fader + SF Mono dB, "+" adds to an available bus), Output picker (Master or a bus), Pan knob, then the long-throw volume fader beside its live meter, an SF Mono glowing dB readout, and the Mute/Solo/Arm row. Buses drop arm/sends/output; master is fader + stereo meter only (no master insert chain in the domain model yet). **Control colors**: volume faders + dB readouts are cyan (matching `MasterVolumeFader`); pan + send controls are neutral white (no semantic claim); **Mute = red** (destructive/removes signal), **Solo = cyan** (active state), **Arm = amber with a slow breathing halo** (record). Knobs/faders are Canvas-drawn with grab-relative drag (⌥ = fine), double-click resets to unity/center, and redraw is value-driven (no per-frame TimelineView allocation). Meters reuse `SegmentMeter` fed from `store.trackMeters` / `masterMeter`, so they animate live during playback.

## Rules

1. Dark theme is the product; light theme is out of scope until M9+.
2. No stock gray AppKit controls in the main window — custom or restyled.
3. One accent color per meaning; never reuse cyan for anything but playback/activity.
4. Animation: 120–200ms ease-out for state; meters/positions animate continuously, chrome never bounces.
5. Text contrast ≥ 4.5:1 for labels; readouts may glow but must pass at core color.
6. Beginner test: every panel understandable by a newcomer in Simple mode without a manual. If a label needs a manual, rewrite it.
